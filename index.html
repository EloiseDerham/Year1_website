Week 9 - Forms

posted on November 27, 2019

Forms are used within a webpage to make it interactive for the user, it allows the website provider to extract and collect data from the users. E.g.

  • Contact details
  • Login details
  • Keyword searches
  • Payment details for ecommerce sites
  • Surveys
  • Uploading user content to social media

HTML for a form is written using the following elements:

  • <form>
  • <fieldset>
  • <legend>
  • <label>
  • <input>
  • <textarea>
  • <button>

These are the basic form elements which can them be styled using CSS

The first step in creating a form is to use the <form> element beneath the navigation and before the footer, making sure to give it an id attribute so that it can be styled independently with CSS.

Within in the form element we use the following elements:

Text type input
allows the user to enter text into boxes
labels
text that tells the user about different form features
Radio buttons
a type of button where only one answer can be selected
fieldset and legend
a box and a caption for different parts of the form

Within a form you have input and label elements, input allows the user to input text while a label describes to the user the data they must enter, these are naturally inline elements , so within the CSS we must define them as block elements for them to display how we wish.

When writing html for a form the “for” attribute within the label element links to the “id” attribute within the input element.

The “value” attribute can be used to give the user an idea of how to fill out the box.

When creating a textarea you must define within the CSS how big you would like it to be using % or em.

Just like label and input for text, radio buttons must be associated in a group with the same value for the name attribute in order to work. You can use the Boolean attribute checked to pre chose one of the radio buttons options.

The submit input element has two possible actions either submitting or resetting the form, at the moment our submit button has no use as we are not connected to a database to submit any data too. I would like to make a reset button that allows the user to clear all from there form.

Extra Reading

Within my form I have used the button element along with the type and value of reset in order to create a button that allows the user to reset the entire form

Further Form elements include:
  • <select> element allows you to create a drop-down menu using <option> to define an option which can be selected.
  • HTML5 <datalist> element specifies a list of pre-defined options for an <input> element.
Further Form attributes include:
  • The multiple attribute allows the user to select more than one value.
  • Read only attribute specifies a field that cannot be edited only read, will not allow the user to edit it.
  • Size attribute specifies the size in character for the input field, just as the maxlength specifies the maximum length of input aloud.
  • The autocomplete attribute allows a form to automatically complete certain input based on a user’s previous inputs on other sites. It works for text, search, url, email, password, datepickers, range and color.
  • The pattern attribute specifies a regular pattern that the inputs value is checked against. It works for text, search, URL, tel, email, and password, this helps to validate a form before submission letting a user know what they have gotten wrong. you can use the global title attribute to describe the pattern to help the user.
  • The autofocus attribute directs focus to a certain field as the page loads.
Further Form types include:
  • The type password can be used to create a box which masks the users input.
  • The New input type file lets the user select and upload a file
  • Input type tel is used for fields that should contain a telephone number and can be used in conjunction with the pattern attribute to specify the pattern the telephone number must follow.

I have used many of these within my own form page.

Form validation

As I learnt from the tutorial it is possible to create some simple form validation using html such as

  • placeholder text showing the user how to fill in a form, using the attribute placeholder.
  • The require attribute to should users which data must be inputted. An error message will be shown if these fields are left blank
  • The email and URL types are validated by the browser to check the format.

Week 8 - CSS3 responsive layout

posted on November 20, 2019

I have used flexbox layout for my tutorial page using one column for smaller screens and two columns for bigger screens. The flexbox model allows elements to be laid horizontally and vertically. I separated the smaller and bigger screen layout using media queries.

For a media query you must set the breakpoints, these are the viewport width at which the columns change. Testing should be undertaken on different devices to check that your min/max width is correct. Media queries are always written after default CSS properties for a page.

Within my media query for my learning journal page I have used display:none to get rid of the nav bar when the page is on a small screen, such as a phone. This improves visibility and makes the page easier to use on smaller devices.

Web page content is displayed in HTML order unless CSS dictates otherwise. One way is by using the float property in CSS , and element will keep floating until it is told to stop by the CSS.

Position element can also alter the flow of HTML, you can use position:relative , position: absolute , position:fixed and position sticky.

Week 6 - Responsive web

posted on November 30, 2019

Before using the CSS3 grid layout out it is a good idea to sketch out your designs so that you know where you would like all of your components to be placed within your page. These are some paper wireframes that I created for both my tutorial and learning journal page:

It might also be a good idea to create different wireframes for different screen widths , for example for if you would like a certain element such as the nav bar to disapear on smaller screens, you could sketch a one column layout design.

The first step to creating a responsive web page is to add the viewport meta tag to the <head> of your HTML document. This ensures that the browser displays the full web page when used on a mobile device.

A good way to lay out your CSS is as follows
  1. Base styles – apply to all versions of the same web page, no matter the viewport width.
  2. Grid Layout rules – sets up the structure for different viewport widths.
  3. You can then write targeted CSS for different screen widths.

You can specify different areas of your grid by using the grid-area property, which can then be used to layout your CSS in the desired column layout.

Rules for grid layout:
  • Every cell filled
  • Repeat name to span more than one cell
  • Use . to create an empty cell
  • Areas should be rectangular
  • Areas cant be repeated in more than one place

It is good practice to Design websites with a mobile first approach.

Display inline and display block can both be used to create different menu types using lists from your HTML. I have used display inline to create a horizontal menu, and display block to create a vertical menu.

As well as the layout of pages media such as images should also be made flexible. To do this firstly all width and height attributes should be deleted from your HTML document, and image max-width should be set to 100% within your CSS.

This will ensure that the images max-width is 100% of its parent element, meaning that as the page changes size the image will change its size in proportion with its parent. for this reason, it is important that when the image is inserted into the website it is already at its largest size and highest resolution.

As you design a responsive website you should make sure to test it on many different devices to ensure the layout is correct across the board. The website that you can use to do this is: http://mobiletest.me/

Extra Reading

Using the fr element with grid to make it flexible. fr represents one fraction of the available space in the grid container.

Grid-column gap is used to create gaps between columns, grid-row gap between rows and grid-gap to set both at once. this can help with layout and design of a webpage, making certain sections easier to see/read. Gaps can be set to be any length unit or percentage but cannot be set to be a fraction.

You can use repeat to write shorter more precise code, repeat (3, 1fr); will create 3 1fr tracks.

Implicit vs explicit grid:

Explicit grid
is created using only gridtemplatecolumn or gridtemplaterows.
Implicit grid
is created when content is placed outside of that grid. implicit grids are given a default size based on there content size, to change these automatic setting you can use grid-auto-rows and grid-auto-columns.

The minmax function lets you set a min and max function for your tracks to prepare for the unpredictability of content size.

Extra Reading

The box model:

To create layouts with CSS you have to understand the box model as everything in CSS has a box around it. The way a box behaves in terms of page flow is defined by these two characteristics.

Block boxes

  • Fills all space available
  • Will break into a new line
  • Width and height properties are respected
  • Using padding, margin and border will cause other elements to be pushed away from the box
  • Elements such as headings and paragraphs all use block as default

Inline boxes

  • Not break into a new line
  • Width and height properties do not apply
  • Padding, margin and borders will apply but not cause inline boxes to move away.
  • Used with elements such as <a> <span> <em> and <strong>.

To change the type of box applied to an element the display property can be used using values block and inline. I have used display and block to style both nav bars on my page.

One of the things that can be edited with the box model is backgrounds and borders, as well as targeting different text directions.

Selectors – target elements in your HTML code by matching the name of the selector with an element in HTML. Selectors are organised in a logical order, starting by following the order that elements are written in HTML with the <body> element first.

It is important to order your CSS properly as CSS using the Cascading rule, where different rules are considered more or less important depending on where they come in the code. This can lead to certain rules in your CSS being overridden by code written later in the document.

Type selectors
match the name of the selector with an HTML tag.
ID selectors
match the name of the selector with the value of an id attribute, a specific id attribute can only be used once in an HTML document, it is unique.
Class selectors
match the name of the selector with the value of the class attribute, the same class attribute can be used many times within one document.

When using a selector in CSS you can then declare what properties and values that element will be presented with in your browser.You can add properties such as font, color , text size , font family, etc.

Extra Reading

The universal selector * ,selects everything in the document , any rule given after the asterisk will be applied to all elements. The universal indicator is only used in certain situations such as making selectors easier to read , stopping any confusing code.

Cascade model in more depth:

The basics of the cascade model is that when two rules apply that have equal specificity the one that comes last in the CSS in the one that is used. An element is also less specific if it selects all elements of that type on the page, it is more specific if it refers to a class or id attribute.

Inheritance in CSS:

If you set certain rules to an element every element inside it will also follow the same rules unless different values have been applied directly. Not all properties inherit, such as width.

CSS provides four special universal property values for controlling inheritance:

Inherit
sets the property value applied to be the same as the parent of the element
Initial
sets the property to be the same as that of the browsers default style sheet
Unset
resets the property to its natural value

I also learn tabout extra selectors that can be used in oreder to alter inheritance properties:

Child selector
Matches an element that is a direct child of another.
Desendent selector
Matches an element that is a descendent of another specified element (not just a direct child of that element.
Adjacent sibling Selector
Matches an element that the next sibling of another.
General sibling selector
Matches an element that is a sibling of another, although it does not have to be the directly preceding element
HTML includes

Can be used in import part of an HTML code that can be imported into any of your web pages, allowing for consistency in user experience. Gets rid of constant editing of repeated content on every new page. Can be achieved using SSI, PHP or Java.

Extra reading:

As I have stressed previously it is extremely important to make your webpage as accessible as possible, one of the reasons for this is so that blind people are able to make sense of your page. By using the right elements, ordering your HTML properly, and adding descriptions to any images or media inserted a screen reader (used by the blind) can make sense of your page and convert it into coherent text for the listener to understand.

I also read more into Hyperlinks and how they work. To understand how a hyperlink works you first need to understand URLS (uniform resource locators) and file paths.

URL
a string of text and defines where something is located on the web, using paths to find files.
Paths
specify where in the filesystem the file you are interested is located.

Links work in a cascading style, for example if you wanted to link to a file that was in the same directory as your current file, e.g. index.html is in the same file as stylesheet.css, you just need to specify the file name. However, if the file was in a subdirectory you would have to move down into said directory before signifying the file you want to link to. To go back up into parent directories ‘..’ can be used + the directory and file name you are looking for.

tips for good link wording:
  • Be specific and precise about where the link is taking you
  • Don’t repeat URLs as part of the link text
  • Don’t say link or links to in the link text it is unnecessary and can draw the user’s attention away from the actual link
  • Don’t let multiple copies of the same text be linked to different places, as it can cause problems to screen reader users.

Using web standards allows websites to be accessible and low cost, making sure they keep on functioning as traditional browsers evolve.
